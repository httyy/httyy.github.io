Re：从零开始的Hexo+Next+Gitpage搭建博客

Posted on 2020-06-15 Edited on 2020-06-16

一直想写博文，来记录自己的学习和生活，但奈何一直没要找到好的平台，CSDN太臃肿，不够简洁，广告还多，博客园的界面又太丑，其他各种各样的博客平台看的我是眼花缭乱，到底选哪个呀！干脆自己搭一个吧，但自己搭的成本太高，需要买域名，云服务器等等，而且还需要具备一定的开发能力。直到在网上找到了使用gitpage搭建博客，真是如获至宝，于是便有了这篇博文。这篇博文的目的一是来记录自己搭建博客过程中学到的各种知识，二是也给想要自己搭建博客的小伙伴一个参考，让一些小白也可以有属于的博客。因此，本博文会从最基础知识开始讲解，详细的记录本博客的搭建过程。当然，由于本人的知识有限，写的过程中难免会出现错误，希望各位大佬可以高抬贵手，轻喷，并指出错误的地方，本人一定虚心接受并改正。那么，就让我们开始吧，let’s go！

一、技术简介

首先先简单介绍一下搭建博客中所有使用的一些技术。如果你已经了解这些技术或对这些技术并不关心，可以直接跳过。

1、Nodejs

Node.js是一个基于 Chrome V8 引擎的 JavaScript 运行环境。JavaScript本来是用来编写网页前端的，而Node让JavaScript可以用来开发网页后端。当然，这些并不是我们的关注重点，我们只需要知道一件事就好了，就是Hexo是基于Node的，因此需要使用Hexo就必须先安装Nodejs。

2、Hexo、Next和Markdown

Hexo是一个基于nodejs的静态博客网站生成器，它可以在几秒内，将我们的文章内容，渲染成静态网页，然后我们就可以将网页发布在网上，就成为一个博客啦。当我们写新的文章或对文章要进行修改时，只需要在完成后，重新渲染，重新发布网页即可。Hexo支持使用Markdown来解析文章，即我们Markdown写文章，Hexo就可以渲染成为网页。

Next是Hexo的一个主题插件，Hexo有很多主题插件，使用不同的主题插件可以让网页的前端展示有不同的样式，而其中Next是Github上Star数最多的主题，因此我选择使用Next主题。当然你也可以选择自己喜欢的主题插件。

Markdown是一种轻量级标记语言，它通过一些简单的标记语法，就可以让普通的文本内容具有一定的格式。Markdown编写的文章可以轻易的导出HTML、Word等文档格式。因此，使用Markdown可以使我们不必太关注文章的格式，将跟多精力放在内容上。Markdown使用起来非常方便，因此除了进行博客书写外，平时做笔记也可以使用Markdown。

3、Github和Gitpage

Github是目前来说影响力最大的代码托管网站，是一个面向开源和私有软件项目的托管平台。在github上，有大量的开源软件作者和开源代码。我们除了可以在github上发布网站外，也可以托管自己的开源代码或查找一些开源软件。

Gitpage是github提供的一个用户托管静态网页的功能，它可以使我们的静态网站部署在github的服务器上，并给分配一个域名。这样，就不需要我们自己去买云服务器和域名了，而且也不需要进行网站备案，非常方便。

二、环境搭建

下面进行发布博客所必要的环境搭建，因为博主使用的是windows系统，因此环境搭建的过程都是基于windows的，后面有时间会补一下linux系统的环境搭建。

1、安装nodejs

Nodejs下载地址为：http://nodejs.cn/download/，目前最新的版本为v14.4.0，需要windows8及以上版本，但14版本目前有bug，使用hexo会报错，因此这里选择了更为稳定的版本v12.18.0。下载mis文件，并进行安装，除了安装路径可以自己指定外，其选择默认即可。

验证安装成功，在windows的cmd中输入如下指令，查看nodejs版本和npm版本：

node -v
npm -v

2、安装git

Github的代码托管是使用git的，因此需要安装git，下载地址为：https://git-scm.com/download/win，目前最新版本为2.27.0。

安装git，同样是除了安装路径可以自己指定外，其选择默认即可，安装完成后验证安装：

git --version

3、安装hexo和next

在windows的cmd中输入如下指令：

npm install -g hexo-cli

验证安装完成，查看hexo版本，博主安装的版本为3.1.0：

hexo -v

由于npm安装使用的是国外的源，因此如果安装超时的话，可以换成阿里的源，就可以正常安装了，命令如下：

替换源：
npm config set registry https://registry.npm.taobao.org

查看源：
npm config list

初始化hexo，创建一个用于存放网页源文件的文件夹，并在cmd中切换到该文件夹中，输入如下指令：

hexo init

初始化，将hexo自带的默认页面渲染成静态页面：

hexo generate #简写为 hexo g

渲染完成后，启动本地服务器：

hexo server #简写为 hexo s

成功后，就可以在本地访问我们生成的网页了，在浏览器中输入如下地址：

localhost:4000

如果看到如下页面，就代表成功了：

下面我们来安装next主题，下载速度非常的慢，博主下载的速度只有几kb的速度：

git clone https://github.com/theme-next/hexo-theme-next themes/next

如果上面的方法下载失败的话，可以直接去github上面下载项目的原文件，地址是https://github.com/theme-next/hexo-theme-next，速度会快很多，但github的网站在国内速度也非常慢，有时候甚至还登不上去，因此需要使用github访问加速（也可以科学上网），原理是修改系统的host文件，绕过国内的dns解析，直接访问github的CDN节点。过程如下：

使用dns解析网站：https://www.ipaddress.com/，查询如下如下地址的ip地址：

github.com
assets-cdn.github.com #会有多个地址，使用一个ping的时延最小的ip
github.global.ssl.fastly.net

然后，根据查询的ip，修改本机host文件：

文件路径：c:\Windows\System32\drivers\etc\hosts

我添加的：
140.82.113.4 github.com
185.199.108.153 assets-cdn.github.com
199.232.69.194 github.global.ssl.fastly.net

然后，在cmd中清空dns缓存，命令如下：

ipconfig /flushdns

然后进入github的next项目页面中，点击如下图中的Download ZIP按钮，下载项目源文件，让后解压文件到我们的hexo文件夹中的themes中，并改名为next：

下载完next主题后，需要在hexo的配置文件中修改默认主题：

文件路径：blogpath\_config.yml #其中blogpath是我们创建的博客文件

将其中：theme: landscape
修改为：theme: next

然后重新渲染并启动我们本地的网页，并在浏览器中访问，如果重新显示如下页面就代表成功了：

就此，我们的环境搭建就算完成了。

三、发布网站

虽然我们的博客现在还没有内容，但已经可以在本地正常运行了，下一步，我们需要将本地生成的静态页面托管到gitpage上，使我们的博客可以在互联网上也可以访问。

1、创建gitpage

首先，你需要注册一个github账号，网址为https://github.com/，然后新创建一个仓库，如下图：

创建仓库的命名是强制性的，具体如下：

<username>.github.io #<username>为你的github账号名

2、部署博客

创建仓库完成后，由于里面还没有任何东西，还不能访问，需要将我们的本地网址部署到gitpage上，首先需要在hexo配置文件中修改部署信息：

文件路径：blogpath\_config.yml #其中blogpath是我们创建的博客文件

找到如下内容：
# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: ''

修改为：
# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: https://github.com/<user>/<repository>.git #其中<user>为github账号名，<repository>为我们创建的gitpage仓库名

修改完成后，输入如下指令进行部署：

hexo deploy #简写hexo d

但这里报错了，报错信息如下：

ERROR Deployer not found: git

这里是由于没有安装git部署的插件，输入如下指令进行安装：

npm install hexo-deployer-git --save

重新部署：

hexo d

这里需要输入github的账号和密码，验证正确后，就完成了网站的部署，这是我们查看gitpage的仓库，就会发现我们的页面已经在仓库里了，这是就可以通过下面的链接访问我们的博客了：

https://<username>.github.io #<username>指github的账号名

3、使用ssh连接github

目前，我们已经完成了网站部署的整个流程（虽然网站还没有内容），但我们每次重新部署我们的网页时，都需要输入github的账号和密码，这样显得非常麻烦，那有没有使我们更方便部署的方法呢？答案是有的，就是使用ssh key进行安全连接，当配置完成后，我们的配置的那台电脑就可以不需要账号密码来操作github的仓库了，具体配置过程如下。

首先，我们需要配置git的账户信息，在cmd中输入如下指令：

git config --global user.name "<user>" #<user>为你的github账号
git config --global user.email "<email>" #<email>为你的github注册邮箱

然后生成ssh key，首先打开git bash，这个是我们安装git自带的工具，输入如下指令：

ssh-keygen -t rsa -C "<email>" #<email>为你的github注册邮箱

执行成功后，会在下面的路径下生成两个文件：

路径：C:\Users\<user>\.ssh #<user>是你电脑的用户名

文件：
id_rsa
id_rsa.pub

然后登陆github账号，在setting中的SSH and GPG keys选择中，点击New SSH key，具体如下图：

然后将id_rsa.pub中的内容复制到key中，点击Add SSH key：

验证ssh key配置成功，在git bash中输入：

ssh -T git@github.com

看到这条：Are you sure you want to continue connecting (yes/no/[fingerprint])?
输入yes

当出现如下文字就代表成功了：

Hi <user>! You've successfully authenticated, but GitHub does not provide shell access.

然后修改hexo的配置文件：

文件路径：blogpath\_config.yml #其中blogpath是我们创建的博客文件

将：
repo: https://github.com/<user>/<repository>.git #其中<user>为github账号名，<repository>为我们创建的gitpage仓库名
修改为：
repo: git@github.com:<user>/<repository>.git #其中<user>为github账号名，<repository>为我们创建的gitpage仓库名

重新测试部署网页：

hexo d

成功！之后部署就再也不需要在输入账号密码了。至此，我们已经完成了这个博客部署的整个过程，后面我们会学习怎么写博文，怎么自己diy我们的博客网站。
